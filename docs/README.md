# WOL-Mini-USB 项目

## 一、项目概述

**WOL-Mini-USB** 是一个**即插即用、零依赖**的远程唤醒/睡眠解决方案。

通过USB HID技术，实现了无需PC客户端软件、无需私有服务器、无需网络唤醒配置的远程控制体验。

## 三、方案说明

### 3.1 工作原理

```
┌─────────────┐     WebSocket      ┌─────────────────┐
│  手机/电脑  │ ───────────────── → │  公共MQTT服务器 │
│  Web控制页  │     (wss://)        │ broker.emqx.io  │
└─────────────┘                     └─────────────────┘
                                           │ MQTT
                                           ↓
                                    ┌─────────────┐
                                    │    ESP32    │
                                    │  WiFi + USB │
                                    │    HID      │
                                    └─────────────┘
                                           │ USB线
                                           ↓
                                    ┌─────────────┐
                                    │     PC      │
                                    │  唤醒/睡眠  │
                                    └─────────────┘
```

### 3.2 核心技术

| 技术 | 说明 |
|------|------|
| **USB HID** | 模拟键盘发送Sleep键，唤醒睡眠PC或触发PC睡眠 |
| **公共MQTT** | 使用broker-cn.emqx.io公共服务器，无需自建 |
| **静态Web** | 纯HTML+JS，可部署到任何静态托管服务 |
| **Token验证** | 本地验证（ESP32端），保护设备安全 |

### 3.3 使用流程

```
1. ESP32插入PC USB口
2. 手机连接WiFi热点配网（热点名：WOL-Mini-USB）
3. 设置设备ID和Token
4. 打开Web控制页面，输入ID+Token
5. 点击"唤醒"或"睡眠"按钮
```

## 四、项目优势

### 4.1 对比传统方案

| 对比项 | 传统WOL方案 | 本方案 |
|--------|-------------|--------|
| PC客户端 | 需要安装 | **不需要** |
| 服务器 | 需要搭建维护 | **不需要**（公共MQTT） |
| 网络唤醒 | 需配置BIOS+路由器 | **不需要** |
| 睡眠控制 | 需PC客户端软件 | **不需要** |
| 部署成本 | 服务器+维护成本 | **零成本** |
| 使用门槛 | 技术配置复杂 | **即插即用** |

### 4.2 核心优势

| 优势 | 说明 |
|------|------|
| ✅ **零依赖** | 无需PC软件、无需服务器、无需复杂配置 |
| ✅ **即插即用** | ESP32插入USB口即可使用 |
| ✅ **零维护** | 公共MQTT服务器，无需运维 |
| ✅ **低成本** | ESP32硬件成本约20元，无后续费用 |
| ✅ **跨平台** | 手机/电脑/平板均可控制，不依赖同一WiFi |
| ✅ **安全性** | Token本地验证，无云端存储 |

## 五、解决的问题

### 5.1 用户痛点

| 痛点 | 传统方案问题 | 本方案解决 |
|------|-------------|------------|
| 远程唤醒PC | 需配置WOL、BIOS设置、路由器端口 | USB插入即可唤醒 |
| 远程睡眠PC | 需安装PC客户端软件、保持运行 | USB HID一键睡眠 |
| 公网访问 | 需搭建服务器、域名、SSL证书 | 公共MQTT，静态Web托管 |
| 运维成本 | 服务器费用、维护时间 | 零运维、零费用 |
| 技术门槛 | 网络配置复杂，普通人难上手 | 配网界面傻瓜式操作 |

## 六、技术细节

### 6.1 ESP32固件功能

| 功能模块 | 说明 |
|----------|------|
| WiFi配网 | 热点模式 + Web界面扫描WiFi |
| USB HID | SystemControl发送STANDBY命令 |
| MQTT通信 | 连接公共broker，订阅控制Topic |
| Token验证 | 本地验证，拒绝无效Token |
| 心跳上报 | 每30秒上报在线状态 |
| 长按BOOT | 进入配网模式（不清空配置） |

### 6.2 Topic设计

```
控制Topic: wol-mini/{device_id}/control
状态Topic: wol-mini/{device_id}/status
```

消息格式：
```json
// 控制消息
{"action":"wake", "token":"12345678", "timestamp":12345}

// 状态消息
{"device_id":"mypc-150C", "status":"online", "ip":"192.168.1.100"}
```

### 6.3 Web技术栈

| 技术 | 用途 |
|------|------|
| HTML5 + CSS3 | iOS风格毛玻璃UI |
| MQTT.js | WebSocket连接公共MQTT |
| LocalStorage | 自动保存设备配置 |

## 七、部署指南

### 7.1 固件烧录

```bash
cd firmware
pio run -t upload
```

### 7.2 Web部署

**方式一：静态托管（推荐）**
- 部署到阿里云、腾讯云OSS
- 或部署到自有服务器Nginx

**方式二：本地访问**
- 直接在浏览器打开web/index.html

### 7.3 注意事项

⚠️ **HTTPS站点需使用wss://连接MQTT**
```
ws://  → 仅HTTP环境可用
wss:// → HTTPS环境必须使用（端口8084）
```

## 八、限制说明

| 限制 | 说明 |
|------|------|
| 唤醒范围 | 只能唤醒S3睡眠状态，无法唤醒关机/休眠 |
| USB依赖 | ESP32需一直插在PC USB口 |
| 网络要求 | ESP32需连接WiFi（2.4GHz） |
